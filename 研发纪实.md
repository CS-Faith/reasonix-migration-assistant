# 研发纪实 / Development Journal

## 第一阶段：理解新版数据格式 / Phase 1: Understanding the Data Format

### 坑 #1：文件名格式 / Filename Format

0.53: `desktop-YYYYMMDDHHMM-N.jsonl`，新版: `YYYYMMDD-HHMM00.000000000-deepseek-v4-flash`。

### 坑 #2：BranchMeta 字段名 / Field Names

新版使用 **snake_case** JSON（`topic_id`, `workspace_root`, `created_at`），不是 camelCase。

### 坑 #3：meta 文件名 / Meta Filename

0.53: `.meta.json`，新版: `.jsonl.meta`（`BranchMetaPath = sessionPath + ".meta"`）。

## 第二阶段：会话不显示 / Phase 2: Sessions Not Appearing

### 坑 #4：ConvertTo-Json 双反斜杠 / Double Backslash

PowerShell `ConvertTo-Json` 输出正确 JSON，不需要再手动转义。

### 坑 #5：JSON 被覆盖 / JSON Overwritten on Startup

新版启动/关闭时读写 `desktop-tabs.json` 和 `desktop-projects.json`，外部修改会被覆盖。 / The new version reads/writes these files on startup/shutdown—external edits are overwritten.

### 坑 #6：侧边栏幽灵条目 / Ghost Sidebar Entries

`orderedTopicIDs()` 遍历 `titleMap` 所有 key，不只是 `globalTopics`。128 个条目全被渲染。 / `orderedTopicIDs()` iterates ALL keys in titleMap, not just globalTopics.

## 第三阶段：BOM 之痛 / Phase 3: The BOM Problem

### 坑 #7：PowerShell UTF-8 BOM

PowerShell `Set-Content -Encoding UTF8` 写入 BOM（0xEFBBBF），Go `json.Unmarshal` 不跳过，导致 meta 解析失败 → 会话打不开。 / Go's json.Unmarshal does not skip BOM → meta parsing fails → sessions can't open.

### 坑 #8：loadTelemetry 的 requestCount 门槛

`requestCount` 必须 ≥1 才加载 telemetry。填 0 导致全部指标不显示。 / Must be ≥1; 0 means all metrics hidden.

## 第四阶段：精雕细琢 / Phase 4: Polish

- 迁移标记绕过侧边栏污染 / Migration markers prevent sidebar pollution
