# Reasonix 配置迁移升级助手 (Migration Assistant)

将旧版 Reasonix 0.53 的对话记录、MCP 配置、记忆、Skill 完整迁移到新版 1.X。

## 快速开始

### 场景 [1]：标准升级（C 盘安装版）

1. 确保 1.5 已安装并至少启动过一次
2. 双击 `启动.bat` → 选择 `【1】`
3. 启动 1.5，点击【历史对话】查看迁移的会话，可手动将其打开并继续对话

### 场景 [2]：便携版升级（U 盘/同步盘）

1. 将 `启动.bat` 和 `Migrate-053to15.ps1` 复制到便携版 Reasonix 根目录
2. 在便携版目录内双击 `启动.bat` → 选择 `【2】`
3. 启动 1.5，点击【历史对话】查看迁移的会话

### 预览模式

选择 `【3】` 预览迁移内容，不实际写入任何文件，可手动将其打开并继续对话

## 迁移内容

| 数据类型 | 说明 |
|---------|------|
| MCP 配置 | JSON → TOML `[[plugins]]` 格式 |
| 对话记录 | 重命名 + 四件套（jsonl/meta/telemetry/ckpt） |
| 运行指标 | token 用量、缓存命中率、费用、依赖文件 |
| 记忆 | memory/ → projects/<slug>/memory/ |
| Skill | 1.5 原生支持，无需迁移 |

## 效果

- ✅ 侧边栏不受影响（保持 1.5 原生内容）
- ✅ 历史对话列表显示全部迁移会话
- ✅ 每条会话保留原始对话标题
- ✅ "打开会话"可继续未完成的对话
- ✅ 对话指标（token/缓存/费用/依赖文件）已迁移

## 注意事项

- 迁移前自动备份 1.5 数据到 `%APPDATA%\reasonix.backup-<时间戳>\`
- MCP 配置仅在 1.5 的 `config.toml` 尚无 `[[plugins]]` 时自动添加
- 重复运行不会重复迁移已有会话（跳过已存在的文件）
- 打开迁移的会话后，tab 标题显示 "Global" 是 1.5 的已知限制，不影响使用。可手动改名成你想要的。
