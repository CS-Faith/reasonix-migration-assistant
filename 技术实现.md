# Reasonix 配置迁移升级助手 — 技术实现文档

## 概述

Reasonix 配置迁移升级助手将 0.53 版本的数据迁移到新版 1.X。

支持两种场景：
- **场景 [1]**：标准安装版 0.53 升级到 1.5（C 盘数据）
- **场景 [2]**：便携版（PortaKit）0.53 升级到 1.5

## 架构

```
启动.bat
  ├─ 场景选择（[1]标准/[2]便携/[3]预览）
  └─ 调用 Migrate-053to15.ps1
       ├─ 源检测（C盘扫描 / 便携自定位）
       ├─ [1/4] MCP 迁移
       ├─ [2/4] 会话迁移
       ├─ [3/4] 记忆迁移
       └─ [4/4] 历史发现准备
```

## 源检测逻辑

### 场景 [1]：标准升级

1. 获取 Windows 登录用户名 `[Environment]::UserName`
2. 检查 `C:\Users\<用户名>\.reasonix\config.json`
3. 如不存在，扫描 `C:\Users\*\.reasonix\config.json`
4. 过滤：排除无 `desktop-*.jsonl` 会话文件的目录（避免选中已升级的目录）

### 场景 [2]：便携版升级

1. 取脚本所在目录 `Split-Path $MyInvocation.MyCommand.Path -Parent`
2. 检查 `<脚本目录>\.reasonix\config.json`
3. 直接使用，不扫描

### 手动指定

通过 `-SourcePath` 参数直接指定源路径。

## 步骤详解

### [1/4] MCP 迁移

```
0.53 config.json               1.5 config.toml
┌─────────────────┐           ┌──────────────────┐
│ "mcp": {        │           │ [[plugins]]      │
│   "server": {   │  ────→   │ name = "server"  │
│     "command":  │           │ command = "..."  │
│     "...",      │           │ args = [...]     │
│     "args": []  │           │                  │
│   }             │           │                  │
│ }               │           │                  │
└─────────────────┘           └──────────────────┘
```

- 解析 0.53 的 `config.json` 中 `mcp` 字段
- 支持两种 0.53 格式：数组 `["name=cmd args"]` 和对象 `{"name": {command, args}}`
- Windows 路径用单引号包裹，以避免 TOML 反斜杠转义问题
- 已有 `[[plugins]]` 配置时跳过（不重复添加）

### [2/4] 会话迁移

```
0.53 sessions/                          1.5 projects/<slug>/sessions/
┌──────────────────────────┐           ┌──────────────────────────────┐
│ desktop-202605270312-1.jsonl    →    20260527-031200.000000000     │
│ desktop-202605270312-1.meta.json    │ .jsonl         ← 对话内容     │
│                                      │ .jsonl.meta    ← 分支元数据   │
│                                      │ .jsonl.telemetry.json ← 指标 │
│                                      │ .ckpt/         ← 检查点      │
└──────────────────────────┘           └──────────────────────────────┘
```

#### 文件命名规则

0.53 命名：`desktop-YYYYMMDDHHMM-N.jsonl`
→ 1.5 命名：`YYYYMMDD-HHMM00.000000000-deepseek-v4-flash`

#### meta 文件 (BranchMeta)

Go 结构体定义自 `branch.go`：

```go
type BranchMeta struct {
    ID            string    `json:"id"`
    CreatedAt     time.Time `json:"created_at"`
    UpdatedAt     time.Time `json:"updated_at"`
    Scope         string    `json:"scope,omitempty"`
    WorkspaceRoot string    `json:"workspace_root,omitempty"`
    TopicID       string    `json:"topic_id,omitempty"`
    TopicTitle    string    `json:"topic_title,omitempty"`
}
```

格式要点：snake_case，时间 RFC3339，空字段用 omitempty。

#### telemetry 文件

```json
{
  "version": 2,
  "readFiles": [...],
  "usage": {
    "promptTokens": ..., "completionTokens": ..., "totalTokens": ...,
    "cacheHitTokens": ..., "cacheMissTokens": ...,
    "requestCount": 1,  // 必须 ≥1 才触发 loadTelemetry
    "elapsedMs": 1,
    "sessionCost": ..., "sessionCurrency": "CNY"
  }
}
```

- 从 0.53 的 `.meta.json` 提取 `lastPromptTokens`, `totalCompletionTokens`, `cacheHitTokens`, `cacheMissTokens`, `totalCostUsd`
- `readFiles` 从 jsonl 中解析 `read_file` 工具调用提取
- `requestCount` 必须 ≥1，否则 `loadTelemetry` 不加载

#### 检查点 (ckpt)

每个用户消息生成一个 `turn-N.json`，格式：
```json
{"turn": N, "time": "RFC3339", "prompt": "前500字符", "msgIndex": 行号, "files": null}
```

### [3/4] 记忆迁移

```
0.53 memory/                    1.5 projects/<slug>/memory/
├── global/                     ├── *.md (全局记忆)
│   ├── *.md          ────→    ├── *.md (项目记忆)
├── <hash>/                     └── MEMORY.md (索引)
│   ├── *.md          ────→
```

- 复制 `memory/global/*.md` 和 `memory/<hash>/*.md`
- 重建 `MEMORY.md` 索引文件

### [4/4] 历史发现准备

目标：迁移的会话出现在历史对话列表，但不污染侧边栏。

```
操作：
1. 给每个会话分配唯一 topic_id
2. 保留原 0.53 对话标题（topic_title）
3. 清空 workspace_root
4. 清空 desktop-topic-titles.json（阻止 orderedTopicIDs 污染侧边栏）
5. 在根 sessions 目录放迁移标记（阻止 migrateLegacySessionsIntoGlobalTopics 重运行）
6. 删除 desktop-projects.json 和 desktop-tabs.json
```

## 1.5 核心机制

### 会话目录结构

```
%APPDATA%\reasonix\
├── config.toml              ← 1.5 配置
├── desktop-projects.json    ← 侧边栏话题列表
├── desktop-tabs.json        ← 打开的 tab
├── global\                  ← 全局元数据
│   ├── desktop-topic-titles.json     ← 话题标题（⚠ orderedTopicIDs 会遍历所有 key）
│   └── desktop-topic-title-sources.json
├── sessions\                ← 根会话目录（migrateLegacySessionsIntoGlobalTopics 扫描此目录）
└── projects\<slug>\sessions\  ← 项目会话目录
```

### WorkspaceSlug

```go
WorkspaceSlug(absPath) = replace("\", "-") + replace("/", "-") + replace(":", "-")
```

例如 `C:\Users\...\global-workspace` → `C--Users-...-global-workspace`

### 侧边栏渲染

`ListProjectTree()` → `orderedTopicIDs(f.GlobalTopics, globalTitleMap)`

⚠ **orderedTopicIDs 会遍历 titleMap 的所有 key**，不仅是 `globalTopics`。所以 `desktop-topic-titles.json` 不能有多余条目。

### 历史对话面板

`ListBranches(dir)` → 扫描 `*.jsonl` → 读取 meta → 返回所有分支。

不依赖 `desktop-projects.json`，独立于侧边栏。

### 打开会话流程

`OpenGlobalTab(topicID)` → `buildTabController()` →
`findTopicSession(dir, topicID)` → 扫描目录 → 匹配 meta.topic_id → 加载会话。

## 编码注意事项

- ⚠ PowerShell `Set-Content -Encoding UTF8` 写入 BOM（0xEFBBBF）
- ⚠ Go 的 `json.Unmarshal` 不跳过 BOM，会导致解析失败
- ✅ 统一使用 `[System.Text.UTF8Encoding]::new($false)` 创建无 BOM 编码
- ✅ 使用 `[System.IO.File]::WriteAllText` 写入文件
- ✅ 读取时检测并跳过 BOM：`if ($txt[0] -eq 0xFEFF) { $txt = $txt.Substring(1) }`
