# 技术实现文档 / Technical Implementation

## 概述 / Overview

将旧版 0.53 的数据迁移到新版 1.X。 / Migrates legacy 0.53 data to the latest 1.X.

支持两种场景 / Two scenarios:
- **场景 [1]**：标准安装版 0.53 升级到新版（C 盘数据） / Standard C-drive upgrade
- **场景 [2]**：便携版（PortaKit）0.53 升级到新版 / Portable upgrade

## 架构 / Architecture

```
0.53配置迁移到1.X.bat        ← 用户入口 / User entry point
  └─ Migrate-053to1X.ps1     ← 核心脚本 / Core script
       ├─ 源检测 / Source detection
       ├─ [1/4] MCP 迁移
       ├─ [2/4] 会话迁移 / Session migration
       ├─ [3/4] 记忆迁移 / Memory migration
       └─ [4/4] 历史发现 / History discovery
```

## 源检测 / Source Detection

**场景 [1]**：扫描 `C:\Users\*\.reasonix\`，过滤无 `desktop-*.jsonl` 的目录。
**场景 [2]**：取脚本所在目录的 `.reasonix\`。

## [1/4] MCP 迁移 / MCP Migration

0.53 `config.json` → 新版 `config.toml` `[[plugins]]` 格式。

Windows 路径用单引号包裹避免 TOML 反斜杠转义问题。 / Windows paths wrapped in single quotes to avoid TOML backslash escaping.

## [2/4] 会话迁移 / Session Migration

```
0.53: desktop-202605270312-1.jsonl
  → 新版: 20260527-031200.000000000-deepseek-v4-flash.jsonl
         + .jsonl.meta      (BranchMeta, snake_case)
         + .jsonl.telemetry.json  (token/cache/cost/readFiles)
         + .ckpt/            (turn-N.json per user message)
```

### meta 文件 / BranchMeta

```go
type BranchMeta struct {
    ID            string    `json:"id"`
    CreatedAt     time.Time `json:"created_at"`
    UpdatedAt     time.Time `json:"updated_at"`
    Scope         string    `json:"scope,omitempty"`
    WorkspaceRoot string    `json:"workspace_root,omitempty"`
    TopicID       string    `json:"topic_id,omitempty"`
    TopicTitle    string    `json:"topic_title,omitempty"`
}
```

字段名来自 `branch.go` struct tag。 / Field names from struct tags in `branch.go`.

### telemetry 文件

从 0.53 `.meta.json` 提取：`lastPromptTokens` → `promptTokens`，`totalCompletionTokens` → `completionTokens`，`cacheHitTokens` / `cacheMissTokens`，`totalCostUsd` → `sessionCost`。

`readFiles` 从 jsonl 中解析 `read_file` 工具调用提取。 / Extracted from `read_file` tool calls in the jsonl.

⚠ `requestCount` 必须 ≥1，否则 `loadTelemetry` 不加载。 / Must be ≥1 or `loadTelemetry` won't load the data.

## [3/4] 记忆迁移 / Memory Migration

复制 `memory/global/*.md` 和 `memory/<hash>/*.md` → `projects/<slug>/memory/`，重建 `MEMORY.md`。

## [4/4] 历史发现 / History Discovery

目标：会话出现在历史列表，不污染侧边栏。 / Goal: sessions in History panel, sidebar stays clean.

```
1. 分配唯一 topic_id / Assign unique topic_id per session
2. 保留原标题 / Preserve original topic_title
3. 清空 workspace_root
4. 清空 desktop-topic-titles.json（阻止 orderedTopicIDs 遍历所有 key）
5. 根目录放迁移标记（阻止 migrateLegacySessionsIntoGlobalTopics）
6. 删除状态文件 / Delete state files
```

## 核心机制 / Key Mechanisms

### 侧边栏 / Sidebar

`ListProjectTree()` → `orderedTopicIDs(f.GlobalTopics, globalTitleMap)`

⚠ `orderedTopicIDs` 遍历 `titleMap` 的**所有 key**，不仅仅是 `globalTopics`。所以 `desktop-topic-titles.json` 必须清空。

### 历史面板 / History Panel

`ListBranches(dir)` 独立扫描 `*.jsonl`，不依赖 `desktop-projects.json`。

### 打开会话 / Opening Sessions

`OpenGlobalTab(topicID)` → `buildTabController()` → `findTopicSession(dir, topicID)` → 匹配 `meta.topic_id` → 加载。

## 编码注意事项 / Encoding Notes

- ⚠ PowerShell `Set-Content -Encoding UTF8` 写入 BOM（0xEFBBBF），Go `json.Unmarshal` 不跳过 BOM
- ✅ 用 `[System.Text.UTF8Encoding]::new($false)` + `[System.IO.File]::WriteAllText`
- ✅ 读取时检测 BOM：`if ($txt[0] -eq 0xFEFF) { $txt = $txt.Substring(1) }`
